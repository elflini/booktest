Chapter 4 데이터 품질 및 분석

4.1 데이터 품질관리

4.1.1 데이터 품질관리의 의의

4.1.1.1 데이터 품질의 정의

  • 데이터가 중시되는 현재 환경에서는 데이터도 매우 중요한 품질관리의 대상이 됨

    • 과거 미국의 주요 기업들 중 고객중심 경여환경의 구축을 위해 고객관계관리(Customer Relationship Management)를 추진하였으나 제대로 성과를 거두지 못하고 실패한 경우가 있음
      • 요인을 분석한 결과, 가장 중요한 요인 중 하나가 고객정보 데이터의 낮은 품질로 나타남
      • 데이터 품질이란 “데이터를 사용하기에 적절한 환경, 즉 적합성, 적시성, 정확성, 완전성, 적절성, 접근 가능성 등을 이르는 말”임
    • 실무적으로 보면, 지식 및 정보와 관련된 업무에 종사하는 사람들이 데이터를 활용하여 업무를 효과적으로 수행하기 위한 데이터의 기대 수준이라고 해석할 수 있음

  • 좋은 품질의 데이터란 데이터의 적시성이 담보되는 가운데 업계 표준을 준수하는 완전하고 일관성 있는 정확한 데이터를 의미함

    • 데이터의 품질을 개선하여 거래 당사자는 생산성을 향상시키는 동시에 거래 속도도 높음으로써 비용을 감소시킴
      • 좋은 품질의 데이터는 거래를 확대시키고 좋은 협업자를 발굴할 수 있는 기회를 제공하며 글로벌 데이터 공유의 토대가 됨
      • 데이터 환경이 급속하게 변화함에 따라 데이터 관리는 공곡ㅇ기관의 영역에서도 중요한 문제로 제기됨(한국보건산업진흥원의 예)

4.1.1.2 데이터 품질관리의 필요성

  • 데이터 품질관리란 업무목적을 효ㅗ과적으로 달성할 수 잇또록 데이터의 품질을 지속적으로 고나리하는 프로세스를 의미함
    • 좋은 데이터 품질은 상품, 용역이나 서비스의 공급 측면에서 볼 때 이러한 공급체계의 효율성을 높이는 최고의 중요 요소임
      • 지속적으로 좋은 품질의 데이터를 유지하면 오류나 오차를 줄일 수 있을 뿐만 아니라, 효율성 자체를 증진시킬 수 잇음
      • 효율성 증진은 상당ㅇ한 비용 절감을 가져올 수 잇으며 이는 고객의 만족으로 이어지며 기업 성과에 매우 긍정적인 영향을 미침
    • 좋은 품질의 데이터는 많은 문제를 해결할 수 있는 해결책을 제시함
      • 만약 데이터의 품질이 제대로 관리되지 않으면 매우 많은 부분에서 경영활동의 제약을 초래함

4.1.2 데이터 품질관리 시스템

4.1.2.1 데이터 품질관리 시스템의 의의

  • 데이터 품질관리 시스템(Data Quality Manangement System)은 데이터의 품질관리를 효과적으로 할 수 있또록 오움
    • 데이터 품질관리 시스템은 신ㄹ회할 수 잇느 ㄴ좋은 데이턱 ㅏ생성되고 공급 체인 내의 원활한 흐름을 통해 원활히 전달, 유지될 수 있도록 하는 내무 프로세스
      • 데이터 품질 프레임워크는 데이터 품질의 개선을 위한 가이드를 제공함
      • 이를 바탕으로 중요한 프로세스 및 조직들이 데잍 ㅓ품질을 개선하고 지속적으로 양질의 데이터 출력을 유지할 수 잇는 환경이 조성됨
    • 데이터 품질관리 시스템은 기존의 시스템 기반 위에 구축할 수 있으므로 초기 비용이 상당히 적게 든다는 장점이 있음

4.1.2.2 데이터 품질관리의 12단계

  • 데이터 품질관리가 효과적으로 실행되기 위해서 주의를 기울여야 할 요소

    • 현존하는 데이터 관리절차가 얼마나 발전된 것인지 판단해야 함
    • 기업의 규모에 따른 적정한 데이터 품질관리 체계가 구축되어야 함
      • 기업의 규모에 비해 지나치게 빈약하거나 과잉인 시스템은 오히려 데이터 품질관리가 효율적으로 이루어지는 데 걸림돌로 작용함
    • 데이터 품질관리 절차의 복잡성(complexity)

  • GS1(2010)은 성공적인 데이터 품질관리를 위해 필요한 필수적인 단계를 12단계로 나누어 설명하고 있음

  • 흔히 발생하기 쉬운 좋지 않은 데이터의 유형을 정리하면 다음과 같다.
낮은 품질의 데이터 예 해결을 위한 노력
결측치나 속성이 알려지지 않은 데이터 결측치의 보완 문제 등에 주의
부정확하고 유효하지 않으며 일관성이 없는 데이터 잘못된 기록이나 정보로 활용되지 않도록 유의
왜곡되고 잘못된 인코딩에 의한 데이터 데이터 왜곡의 이유를 파악하고 이에 대한 처리방법을 논의
잘못된 해석을 초래하는 데이터 데이터 설명을 충분히 숙지
오염된 데이터 데이터클리닝 과정 고려
중요성을 고나과하고 있는 지나치게 많은 양의 데이터 중요성을 지니는 부분 선별

4.1.3 데이터 품질관리 사례

  • 품질관리 시스템을 성공적으로 동비한 공공기관의 사례를 중심으로 데이터 품질관리 결과 얼마나 업무성과가 개선되었는지 살펴봄( 정부 3.0 및 창조경제를 위한 데이터 품질관리 사례집(2013))

4.1.3.1 관세청의 데이터 품질관리 사례

  • 우리나라는 무역 의존도가 높으므로 통관 데이터의 중요성이 매우 강조되고 있음
    • 관세청은 고품질 데이터 확보와 체계적인 품질관리 활동 수행을 위해 중장기 ISP 추진을 통해 데이터 품질관리 로드맵을 수립하고 단계적으로 과제를 추진함
    • 관세청의 데이터는 관세사, 화주, 선사 등의 수출입 관련 민원 입력을 통해 직접 생성되는 특징이 있으며, 이 과정에서 낮은 품질의 부정확한 데이터 입력이 주요 이슈로 제기됨
  • 이러한 문제를 개선하기 위해 관세청은 부정확한 데이터 입력을 방지하기 위한 소프트웨어를 개발함
    • 데이터를 생산하는 관세사, 선사 등 약 1,000명을 대상으로 이를 이용한 입력 오류 및 사전검증 체계 교육을 실시함
    • 부정확한 입력 데이터의 사전검증을 위한 참조 데이터베이스를 구축하고 시범 적용함으로써 입력 통제를 자동화함
  • 중복되어 있고, 부정확하게 등록되어 있는 해외거래처 중복코드 정제로 거래처 정보 정확도를 향상시킴

4.1.3.2 특허청의 데이터 품질관리 사례

  • 지식재산 정보 관련 문제들을 미연에 방지하고, 국민에게 신뢰할 수 있는 지식재산 정보 서비슷를 제공하기 위해서는 최적화된 데이터 품질관리 체계를 구축해 고품질의 특허 데이터를 확보할 필요가 있음

    • 특허청에서는 데이터 품질 향상을 위해 다년간의 노렭ㄱ과 관심을 투자한 결과 특허 데이터 품질관리 체계를 구축하여 정부 기관의 선도적인 역할을 수행함

      • 2007년 ’데이터 품질관리 체계 구축 정보화 전략기획(ISP)’을 실시하였으며, 2008년 데이터 품질관리 1차 사업을 수행하고, 2009년에는 ’데이터 품질관리 체계 고도화’를 위한 2차 사업이 이어짐
      • 2010년에는 데이터 품질관리 안정화와 중요성 인식을 위한 문화 조성을 추진하고, 2010년부터 현재까지는 ‘3세대 특허넷 시스템 구축’ 등을 통하여 데-이터 품질 기반의 시스템을 마련함
      • 2012년에는 ‘특허 데이터 품질관리 체계 선진화’ 사업을 수행해 데이터 관련 중장기 정보화 전략을 수립함

  • 장기적인 사업 수행 덕분에 특허청은 전사적인 특허 데이터에 대한 품질관리의 중시문화 및 의식을 조성할 수 있었음

4.1.3.3 건강보험심사평가원의 데이터 품질관리 사례

  • 평가원은 8만여 요양기관, 연간 13억 건, 45조 원에 달하는 방대한 규모와 다양한 연계기관 간의 데이터 처리를 위해 청구포털, 요양기관포털, 심사평가, DW 등 방대한 정보 시스템을 적극 활용함

    • 축적된 방대한 양의 데이터를 활용하고 품질을 유지하기 위해 품질관리 활동을 지속적으로 운영함
      • 2011년 데이터 품질관리 체계 구축작업과 차세대 정보 시스템 구축사업이 함께 진행되면서, 데이터 표준화에 대한 필요성이 제기되어 차세대 정보 시스템 구축 시 데이터 표준화를 추진함
      • 관련 실무자들이 포함된 데이터 품질개선팀을 구성해 관리 활동을 지속하고 있음

4.2 데이터 분석

4.2.1 데이터 분석의 의의

4.2.1.1 데이터 분석 도입의 배경

  • 최근의 환경변화에 발맞추어 다수의 기업이 데이터 분석의 중요성을 인식하기 시작하였으며 데이터 분석을 통한 최적 의사결정이라는 과제를 기업의 주요 전략 중 하나로 삼고 있음

    • 데이터의 미비가 기업가치에 미치는 영향 그림 삽입

    • 데이터와 데이터 분석이 기업의 의사결정 과정에 큰 영향을 미침

    • 의사결정에까지 이르는 과정에서 데이터가 충분히 확보되지 않아 분석에 문제가 발생하고 있음

      • 데이터 분석을 어떻게 기획하느냐에 따라 발생 가능한 문제를 사전에 차단하고 기업업무 운영 전반의 최적화도 이룰 수 있음

4.2.1.2 데이터 분석 도입의 성공요소

  • 한국데이터베이스 진흥원(2014)에서는 세 가지 측면을 데이터 분석 도입의 성공 요소로 들고 있음

    • 분석 질문의 우선적 정의
      • 무엇을 분석할 것인지 구체적이고 명확하게 정의할 필요가 있음

  • 질문의 우선적 정의(Question First) 방식 그림 삽입

    • 선택과 집중
      • 데이터의 규모가 커지고 형태가 복잡해질수록 선택과 집중이 더욱 중요한 문제로 부각됨
      • 선택과 집중을 기반으로 핵심 분석과정이 빠른 시간 내에 원활히 완료됨으로써 기업의 경쟁력 강화하는 긍정적인 효과를 얻을 수 있고 향후 유사 업무 프로세스에 적용할 수 있음

  • 선택과 집중 그림 삽입

    • 자동화된 분석의 업무 프로세스 내재화
      • 데이터 분석이 의사결정에 필수적인 요소로서 적절하게 활용되기 위해서는 분석이 기업의 의사결정 프로세스의 일부로 자리 잡아야 함
      • 분석을 업무 프로세스에 내재화하여 시간적인 단축을 유도할 뿐만 아니라 분석과 의사결정을 함께 연계하여 생각해 볼 수 있는 통찰력도 지닐 수 있게 됨

  • 분석업무의 내재화 그림 삽입

4.2.2 분석 핵심 문제의 발견

4.2.2.1 분석기획

  • 데이터 분석기획 단계는 기초 재료를 가지고 얼마나 믿을 수 있고 실현 가능한, 성공적인 성과물을 만들 수 있는지 청사진을 제시하고 도면을 설계하는 과정

    • 다양한 업무 유형과 엄청나게 쏟아지는 데이터, 그리고 복잡한 경영의 문제가 얽혀 있을 때, 분석 대상을 얼마나 효과적으로 찾아내고 적시에 분석 절차를 진행하느냐가 결국 기업의 성패를 좌우한다고 할 수 있음

4.2.2.2 접근방법

  • 분석의 핵심 문제를 적절하게 찾기 위해서는 업무 프로세스에 대한 이해가 필수 적임

    • 분석의 핵심 문제를 찾는 일련의 과정을 분석기회의 발견이라 정의하며 세 가지 접근 방식이 있음
      • 전사적인 분석체계 구현을 위해서 인력, 조직, 기술, 관리 전반을 다루는 하향식(톱다운) 접근 방식(top-down approach)
      • 실제적인 업무 프로세스상에서 특정 주제를 찾아내어 분석기회를 발굴하고 주제별 분석기회를 구현해 보는 상향식(바텀업) 경로접근 방식(bottom-up path finding approach)
      • 특정 산업이나 업무별 분석 대상 풀(pool)로부터 벤치마킹을 하여 분석기회를 찾는 접근방식

  • 분석 핵심 문제의 세가지 방법 그림 삽입

4.2.2.3 분석기회의 정의

  • 데이터 분석기회란 주요한 의사결정의 대상이 되는 업무를 정의하고 이 업무를 효과적으로 수행하기 위해서 알아야 할 주요 사항들을 정리하고 찾는 방법을 말함
    • 분석기회를 명확화 구체화하는 과정을 통해야만 적절한 분석에 이르게 되고 정확한 상황판단과 의사결정에 도움을 주게 됨

4.2.2.4 구체적인 분석 질무의 작성

  • 분석기회가 정의되고 이를 통해 기업의 목표가 구체화되었다면 그다음 단계에서 필요한 것이 구체적인 분석 질문을 작성하는 과정임

    • 업무 담당자는 의사결정 단계에서 필요한 정보를 찾아내기 위해 질문을 던지게 됨
      • 분석 질문ㅇ르 잘 만들수록 필요한 정보를 더 많이 확보할 수 있고, 적절한 분석을 진행할 수 있음
      • 분석 질문 방식의 대표적인 예로는 연속 질문 방식과 업무요소별 질문 방식을 꼽을 수 있음

  • 질문을 도출해 내었다면 도출한 질문이 제댈 ㅗ제시된 것인지, 적절한 것인지 평가하고 재검토할 필요가 있음

    • 보조 질문을 가지고 역으로 검토해 가는 방법을 활용
      1. 연속 질문 방식
      2. 업무요소별 질문 방식

  • 이제까지 살펴본 데이터 분석 핵심 문제를 정의하기 위한 전반적인 절차를 정리하면 다음과 같음

  • 연속 질뭉르 통한 필요 분석의 도출 그림 삽입

  • 업무요소별 질문 방식 그림 삽입

  • 분석 핵심 문제를 정의하는 절차 그림 삽입

4.2.3 분석 문제 탐색과정의 구조화 및 구체과

  • 데이터 분석에서 또 한 가지 중요한 사항은 분석의 핵심 문제를 발견하는 과정 자체를 구조화 구체화하는 과정임
    • 데이터 핵심 문제의 정의와 비교해 볼 때, 문석의 핵심 문제를 발견하는 과정을 구조화 구체화하는 것은 상당히 폭넓은 개념임

4.2.3.1 분석문제 탐색과정의 구조화

  • 분석문제 탐색과정을 구조화하기 위해서 유용한 방법은 유저스토리의 작성임(한국데이터베이스진흥원(2014)))

    • 유저스토리란 분석자의 역할, 의사결정 사항, 분석을 통해 이루고자 하는 목표가치로 구성된 정보 집합을 말함

  • 유저스토리의 작성 그림 삽입

4.2.3.2 목표가치의 구체화

  • 유저스토리를 바탕으로 추구하는 목표가치를 구체화함
    • 목표를 측정 가능한 형태인 지표로 정의하는 과정이 필요함

4.2.3.3 데이터 분석 방안의 구체화

  • 데이터 분석을 위한 분석 질문들이 정의되고, 그 대상과 필요한 분석들이 도출되었다면 분석 대상들 간의 관계성을 모형화할 필요가 있음
    • 분석 대상들 간의 상관관계를 평가하여 의사결정에 필요한 각각ㄱ의 요소 간 관계를 구체적으로 살펴볼 수 있음

4.3 요약

  1. 데이터 품질의 사전적 정의는 “데이터를 사용하기에 적절한 환경, 즉 적합성, 적시성, 정확성, 완전성, 적절성, 접근 가능성 등을 이르는 말”이다. 실무적으로 보면, 지식 및 정보와 관련된 업무에 종사하는 사람들이 데이터를 활용하여 업무를 효과적으로 수행하기 위한 데이터의 기대 수준이라고 해석할 수 있다.

  2. 데이터 품질관리란 업무목적을 효과적으로 달성할 수 있도록 데이터의 품질을 지속적으로 관리하는 프로세스를 의미한다.

  3. 지속적으로 좋은 품질의 데이터를 유지하면 오류나 오차를 줄일 수 있을 뿐만아니라, 효율성 자체를 증진시킬 수 있따.

  4. 데이터 분석 도입의 세 가지 성공요소는 첫째, 분석 질문의 우선적 정의, 둘째, 선택과 집중, 셋째, 분석업무의 내재화를 꼽을 수 있따.

  5. 데잍 ㅓ분석기회란 주요한 의사결정의 대상이 되는 업무를 정의하고 이 업무를 효과적으로 수행하기 위해서 알아야 할 주요 사항들을 정리하고 찾는 방법이다. ’

  6. 분석 질문을 분석 대상을 파악하는 데 매우 중요한 과정이라고 할 수 있으며, 이러한 분석 질문을 구체화하는 방법으로는 연속 질문 방식과 업무요소별 질문 방식을 들 수 있다.

  7. 데이터 핵심 문제의 정의와 비교해 볼 때, 분석의 핵심 문제를 발견하는 과정을 구조화 구체화하는 것은 상당히 폭ㄴ럽은 개념이다. 전자가 핵심 문제 정의 자체에 초점을 맞춘 것이라면, 후자의 경우는 궁극적으로 분석의 방향성과 전반적인 절차의 명확화를 목표로 하고 있다.
